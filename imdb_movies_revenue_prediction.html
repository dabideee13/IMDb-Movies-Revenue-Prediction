Movies revenue prediction¶

A Regression, K-NN, SVM application.¶

by Dave Emmanuel Q. Magno and Aries Valeriano

Executive Summary¶

The main goal of this project is to build a prediction model which takes features such as budget, company, country, director, genre, rating, released, runtime, votes, star, writer, year as predictors to predict the gross/ revenue of a movie.

To achieve this goal, data processing up to predictive modelling were performed. Moreover, for reproducibility, syntax are shown with corresponding comments for better understanding about its used.

In the data processing, several steps were performed due to the datasets having categorical variables with too many levels, as well as variable that have missing values. For categorical variables, one hat encoding method were performed. As for the missing value, after visualising its distribution, it was found out that there are too many outliers and as result median imputation was performed over mean imputation since it is more robust to outliers.

In data exploration information about the datasets are further explored. The data type, dimension, and the variables included in it. Moreover, linear relationship between pairs of numerical features including the target feature were investigated. And we found out that budget feature seem to have a moderate positive linear relationship with gross/ revenue. Whereas, votes and runtime only have a little positive linear relationship with gross/ revenue.

In predictive modelling, to find the model that could produced the highest accuracy, different types of regression model such as; multiple linear regression, ridge, lasso, and elastic net were performed. In addition, support vector machine, and k-nearest neighbor models were also performed to compare there predictive accuracy to regression models.

Then, it was found out that the model that produces the highest predictive accuracy is the elastic net regression model. This model let us predict the revenue/ gross of a movie given the following descriptive features; 'budget', 'director', 'runtime', 'star', 'votes', 'writer', 'year', 'dayReleased', 'monthReleased', 'company_Columbia Pictures', 'company_Columbia Pictures Corporation', 'company_New Line Cinema', 'company_Others', 'company_Paramount Pictures', 'company_Touchstone Pictures', 'company_Twentieth Century Fox Film Corporation', 'company_Universal Pictures', 'company_Warner Bros.', 'country_Canada', 'country_France', 'country_Germany', 'country_Others', 'country_UK','country_USA', 'genre_Action', 'genre_Adventure', 'genre_Animation', 'genre_Biography', 'genre_Comedy', 'genre_Crime', 'genre_Drama', 'genre_Horror', 'genre_Others', 'rating_G', 'rating_Others', 'rating_PG', 'rating_PG-13', 'rating_R', 'rating_UNRATED' with 65.35% accuracy.

def get_unique(df: pd.DataFrame) -> dict:
    """Get distinct values or categories in categorical 
    columns."""
    return {col_name: len(set(df[col_name]))
            for col_name in df.columns}

def lump_categories(df: pd.DataFrame):
    """Lumps categories who has lesser counts
    than the threshold to 'Others'."""
    
    def set_threshold(df: pd.DataFrame,
                      percentage: float=0.015) -> float:
        """Sets threshold to be a percentage of the size
        of dataset."""
        return df.shape[0] * percentage   
    
    return df.apply(
        lambda x: x.mask(
            x.map(
                x.value_counts()) < set_threshold(df), 
                         'Others') 
        if x.name in list(df.columns) else x)

def replace_by_threshold(series: pd.Series,
                         threshold: int=9) -> pd.Series:
    """Replaces director, star, and writer values
    by 1 if > 9 and 0 otherwise."""
    
    def map_frequency(series: pd.Series) -> dict:
        """Get frequency for each category and maps
        each labels to its frequency."""
        return series.value_counts().to_dict()
    
    def replace_by_frequency(series: pd.Series,
                             mapping: dict) -> pd.Series:
        """Replaces the labels by its frequency."""
        return series.map(mapping)
    
    mapping = map_frequency(series)
    new_series = replace_by_frequency(series,
                                      mapping)
    
    return np.where(new_series > threshold, 1, 0)

def extract_day(data: pd.Series) -> pd.Series:
    """Given the released date, extract
    only the day."""
    return data.dt.day_name()

def extract_month(data: pd.Series) -> pd.Series:
    """Given the released date, extract
    only the month."""
    return data.dt.month_name()

class pipe:
    def __init__(self, value, func=None):
        self.value = value
        self.func = func
    def __getitem__(self, func):
        return pipe(self.value, func)        
    def __call__(self, *args, **kwargs):
        return pipe(self.func(self.value, *args, **kwargs))
    def __repr__(self):
        return 'pipe(%s, %s)' % (self.value, self.func)

def encode_label(df: pd.DataFrame) -> pd.DataFrame:
    """Encode categories as labels, same with 
    LabelEncoder() by scikit-learn."""
    for col in df.columns:
        df[col] = df[col].astype('category')
        df[col] = df[col].cat.codes
    return df

def final_comparison(models, 
                     X_test: pd.DataFrame, 
                     y_test: pd.DataFrame):
    scores = pd.DataFrame()
    for model in models:
        predictions = model.predict(X_test)
        mae = round(mean_absolute_error(y_test, predictions), 4)
        mse = round(mean_squared_error(y_test, predictions), 4)
        r2 = round(r2_score(y_test, predictions), 4)
        errors = abs(predictions - y_test)
        mape = 100 * np.mean(errors / y_test)
        accuracy = round(100 - mape, 4)
        scores[str(model)] = [mae, mse, r2, accuracy]
    scores.index = ['Mean Absolute Error', 'Mean Squared Error', 'R^2', 'Accuracy']
    return scores

Data Description¶

The data is available at https://www.kaggle.com/danielgrijalvas/movies. This is a three decades worth of movie data (1986-2016), scraped from IMDb using Python. The dataset has the following content:

There are 6820 movies in the dataset (220 movies per year, 1986-2016). Each movie has the following attributes:

  • budget: the budget of a movie. Some movies don’t have this, so it appears as 0
  • company: the production company
  • country: country of origin
  • director: the director
  • genre: main genre of the movie.
  • gross: revenue of the movie
  • name: name of the movie
  • rating: rating of the movie (R, PG, etc.)
  • released: release date (YYYY-MM-DD)
  • runtime: duration of the movie
  • score: IMDb user rating
  • votes: number of user votes
  • star: main actor/actress
  • writer: writer of the movie
  • year: year of release

The task at hand, is to predict the revenue/gross using the rest of the features.

These dataset will be split into training set which contains (4774) samples and testing set which contains (2046) samples. Training set is used for building a model and testing set will be used for assessing the accuracy of the model built from training set.

Some combination of levels of feature rating that seem to have a similar implication also was performed. Furthermore, the presence of missing values were investigated. Conclusion and recommendation¶

The datasets given is a little bit messy, we found out that missing values are present thus, we performed median imputation after performing boxplot that shows a lot of outliers. Moreover, categorical variables have too many levels that needs to be lump and recode. Furthermore, after investigating the relationship between numeric descriptive features and the target feature gross/revenue, it was found out that budget has moderate positive linear relationship with gross moreover, both votes and runtime does have a little positive relationship with gross.

Through employing different type of regression model such as multiple linear regression, ridge, lasso, and elastic net regression we found out that elastic net regression produced a little bit better predictive acccuracy compared to the rest. It provide us a 65.35% predictive accuracy.

Furthermore, by employing support vector machine and k-nearest neighbors models, we found out that k-nearest neighbors regressor has 40.73 accuracy, while support vector machine has -13.67.

Therefore, after comparing all the predictive accuracy we got from the few numbers of models we tried. The predictive model we built that has the highest accuracy is from elastic net regression. With this model, we can predict the gross or revenue of a movie given the following descriptive features; 'budget', 'director', 'runtime', 'star', 'votes', 'writer', 'year', 'dayReleased', 'monthReleased', 'company_Columbia Pictures', 'company_Columbia Pictures Corporation', 'company_New Line Cinema', 'company_Others', 'company_Paramount Pictures', 'company_Touchstone Pictures', 'company_Twentieth Century Fox Film Corporation', 'company_Universal Pictures', 'company_Warner Bros.', 'country_Canada', 'country_France', 'country_Germany', 'country_Others', 'country_UK','country_USA', 'genre_Action', 'genre_Adventure', 'genre_Animation', 'genre_Biography', 'genre_Comedy', 'genre_Crime', 'genre_Drama', 'genre_Horror', 'genre_Others', 'rating_G', 'rating_Others', 'rating_PG', 'rating_PG-13', 'rating_R', 'rating_UNRATED' with 65.35% accuracy.

For the recommendation, during data preprocessing, since there are too many descriptive features we tried to reduce the number of descriptive features using backward elimination procedure and lasso regression however, it does not help the predictive accuracy of the model built getting better. Thus, how about try another method of feature selection such as PCA etc?. Also, in regression modelling, since there are few assumptions that should be met we did some investigation about it in our datasets, then, we found out the datasets failed to meet some of the assumptions. Hence, we tried to meet these assumptions by doing data transformation however, it still didn't improve the predictive accuracy of our regression models. Maybe because even we've done data transformation to normalize it or to meet other assumptions it still didn't meet the assumptions because of the method we used. Thus, we recommend to try transformation method or any other method for other assumptions that are really appropriate for the datasets.
